Como Apontar Horas no Ekyte
Usando o Claude

Com esse skill configurado, você registra seus apontamentos de timesheet com uma única frase — sem navegar na plataforma, sem lembrar de formatos, sem interromper o fluxo de trabalho.

Tempo de configuração~10 minutos (uma única vez)
Validade do token~6 meses
Uso diárioUma frase pro Claude
O que você vai conseguir fazer

Falar pro Claude: "Timer task 9434115 das 09:00 às 10:30 de hoje" — e ele registra automaticamente no Ekyte para você, com confirmação antes de salvar.

Antes de começar

Claude Code instalado

É o programa que vai executar o skill. Peça para quem configurou o seu para instalar também.

Chrome com Ekyte aberto

Você vai precisar do Chrome especificamente para capturar a requisição no DevTools.

Acesso normal ao Ekyte

Você precisa conseguir logar um timer manualmente uma única vez durante a configuração.

Importante: Cada pessoa precisa fazer essa configuração com a sua própria conta do Ekyte. O token é individual — não é possível usar o de outra pessoa, pois ele está vinculado ao seu login.
Parte 1
Configuração inicial

Por que esses passos são necessários?

Para o Claude registrar horas no Ekyte em seu lugar, ele precisa de uma "chave de acesso" da sua conta — chamada token. Essa chave fica dentro das comunicações que o seu navegador faz com o Ekyte quando você usa a plataforma normalmente.

A forma mais simples de obter esse token é usar o DevTools do Chrome — uma ferramenta que já vem instalada em todo Chrome — enquanto você realiza uma ação no Ekyte. O DevTools registra essas comunicações, e dentro de uma delas está o seu token pessoal.

1

Abrir o DevTools no Chrome

  • Com o Ekyte aberto no Chrome, pressione a tecla F12 no teclado
  • Uma janela vai abrir na lateral ou na parte inferior do navegador — isso é o DevTools
  • Clique na aba Network (fica na barra superior do DevTools)
  • Deixe essa janela aberta — não feche
Dica: Se aparecer uma mensagem perguntando sobre o idioma do DevTools, pode ignorar ou clicar em "Don't show again".
2

Registrar um timer qualquer no Ekyte

Agora você vai realizar uma ação normal no Ekyte para que o DevTools capture a requisição com o seu token. O apontamento feito neste passo pode ser qualquer horário — o objetivo aqui é apenas capturar o token para a configuração, não registrar horas de verdade.

  • Acesse qualquer task sua no Ekyte
  • Clique para adicionar um apontamento de tempo (timer manual)
  • Coloque qualquer horário (ex.: 09:00 às 09:05)
  • Salve o apontamento
3

Encontrar a requisição certa no DevTools

Após salvar o apontamento, observe a lista de requisições que apareceu no DevTools. Você vai ver várias linhas — precisa identificar a correta:

O que procurar Como identificar
Nome time-trackings
Tipo xhr
Status 200
Dica: As requisições aparecem em ordem cronológica. As mais recentes ficam no final da lista — role para baixo para encontrá-las.
4

Copiar a requisição

  • Clique com o botão direito do mouse na linha time-trackings
  • Vá em Copy
  • Clique em Copy as fetch

Pronto — o conteúdo foi copiado para sua área de transferência. O texto copiado parece código, mas você não precisa entender o que está escrito. É só copiar e colar no próximo passo.

5

Enviar para o Claude configurar

Abra o Claude Code e envie a mensagem abaixo, substituindo apenas o trecho indicado em colchetes:

Cole essa mensagem no Claude

Quero configurar o skill de timer do Ekyte para registrar meus apontamentos de horas automaticamente pelo Claude. Aqui está minha requisição do DevTools:

[COLE AQUI O QUE VOCÊ COPIOU NO PASSO 4]

O Claude vai:

  • Ler o código copiado e extrair automaticamente seu token e o ID do seu workspace
  • Criar o arquivo do skill com as suas informações
  • Confirmar quando estiver pronto
6

Testar se funcionou

Após o Claude confirmar a configuração, envie a seguinte mensagem para testar. Substitua pelo número de qualquer task sua:

Frase de teste

"Timer task [número de qualquer task sua] das 09:00 às 09:30 de hoje"

Se o Claude responder com "Timer registrado com sucesso" e mostrar o ID do registro — está tudo funcionando.

Parte 2
Como usar no dia a dia

Como instruir o Claude a apontar horas

Você pode usar várias formas diferentes. O Claude entende todas:

Situação O que enviar para o Claude
Apontar pelo ID da task Timer task 9434115 das 09:04 às 10:20
Apontar em outro dia Timer task 9434115 das 14:00 às 15:30 do dia 03/06
Apontar pelo nome da task Timer da task Verificar redes sociais das 08:30 às 09:00
Não lembrar o ID Informe o nome parcial — o Claude busca e confirma antes de registrar

O que o Claude mostra antes de registrar

Antes de confirmar qualquer apontamento, o Claude sempre exibe um resumo para a sua validação. Você confirma e ele registra — não é possível salvar no workspace errado sem ver o resumo antes.

Campo Exemplo
TaskVerificar redes sociais dos clientes
Período08:30 às 09:04 — 02/06/2026
Duração34 minutos
Parte 3
Manutenção

Quando o token vai expirar?

O token tem validade de aproximadamente 6 meses a partir da data de configuração. Quando expirar, o Claude vai informar com uma mensagem de erro. Para renovar, basta repetir os Passos 1 ao 5 desta configuração — o processo leva menos de 5 minutos e precisa ser feito uma vez por semestre.

Erros comuns e como resolver

Mensagem que aparece O que significa O que fazer
"Token expirado" ou erro 401 Seu token venceu Refaça os Passos 1 ao 5
"Task não encontrada" O ID ou nome não correspondeu Confirme o ID no Ekyte e tente novamente
"Horário de fim antes do início" Os horários foram invertidos Corrija e envie novamente
